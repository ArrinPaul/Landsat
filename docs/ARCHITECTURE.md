# Technical Architecture Specification

This document provides a detailed overview of the architectural design and data flow within the Earth Insights Analysis Platform.

## System Overview

The platform is designed as a modular full-stack application leveraging Next.js 15. It integrates specialized geospatial processing with generative artificial intelligence to transform raw satellite data into interpreted environmental intelligence.

## Data Processing Layers

### 1. Geospatial Processing Layer (Google Earth Engine)
The primary data source for satellite imagery is the Google Earth Engine (GEE).
*   **Data Source:** COPERNICUS/S2_SR_HARMONIZED (Sentinel-2 Surface Reflectance).
*   **Computation:** The system performs server-side computations of spectral indices (NDVI, NDWI, NDBI, NBR) across a defined temporal and spatial range.
*   **Output:** Returns time-series data and classified thumbnail imagery for client-side visualization.

### 2. Meteorological and Pedological Layer (Open-Meteo)
Environmental context is provided by integrating weather and soil data.
*   **Weather Data:** Historical temperature and precipitation records.
*   **Soil Analysis:** Real-time volumetric water content and soil classification (Loam, Clay, Sand, etc.).
*   **API Management:** Implements fallback mechanisms and SSL agent configurations for reliable data retrieval.

### 3. Artificial Intelligence Orchestration (Google Genkit)
The AI layer interprets processed data using Large Language Models (LLMs).
*   **Primary Model:** Google Gemini 2.0 Flash.
*   **Fallback Chain:** In the event of rate limiting or service unavailability, the system automatically transitions through a configured provider list: Groq (Llama), Mistral, and HuggingFace.
*   **Flows:** Discrete AI flows are defined for specific tasks such as crop yield prediction, change analysis interpretation, and conversational assistance.

## Software Architecture

### Backend (Server Actions)
The application utilizes Next.js Server Actions for secure, server-side execution of business logic.
*   **Job Management:** Implements an asynchronous job pattern for long-running GEE computations.
*   **Rate Limiting:** A token-bucket inspired rate limiter prevents API abuse.
*   **Security:** Centralized credential management for Google Cloud and AI provider keys.

### Frontend (User Interface)
The client-side application is built for performance and interactivity.
*   **State Management:** Utilizes React hooks for localized state and polling mechanisms for asynchronous job updates.
*   **Visualization:** Data is rendered using Recharts for time-series analysis and customized UI components for metric summaries.
*   **Internationalization:** Implements a context-based localization provider supporting multiple regional languages.

## Security and Authentication

The platform is designed to integrate with Firebase Authentication for user identity management. Currently, the system uses a secure server-side environment for credential storage, ensuring that sensitive API keys for Google Cloud and AI providers are never exposed to the client.

## Machine Learning Strategy

The platform employs a multi-tiered approach to machine learning:
*   **Heuristic Analysis:** Initial classification is performed using scientifically established index thresholds.
*   **Generative Reasoning:** Complex agricultural and environmental predictions are generated by providing LLMs with multi-dimensional data context.
*   **Future Integration:** The architecture is designed to support the transition to specialized regression models and TensorFlow-based classifiers for higher scientific precision.
